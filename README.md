# dynamic-bot-bilibili

Bilibili 动态源插件，用于让 dynamic-bot 订阅 Bilibili UP 主动态和直播状态，并支持解析 Bilibili 链接。

## 功能

- 添加订阅时支持 UID 和用户名搜索。
- 支持二维码登录和 Cookie 登录。
- 可在订阅时自动关注 UP 主。
- 检测新动态。
- 检测直播开播和下播。
- 解析动态、视频、直播间、用户空间和 b23.tv 短链接。
- 链接解析时可下载视频。
- 自动维护运行中的 Cookie。
- 遇到未登录或风控时会暂停轮询，避免持续请求。

## 安装

构建插件：

```powershell
.\gradlew.bat fatJar --offline
```

把生成的 `*-all.jar` 放入主程序的：

```text
plugins/
```

然后启动或重启 dynamic-bot，在后台启用插件。

## 登录

推荐在 Web 后台使用二维码登录。

登录成功后，插件会保存 Cookie。后续运行中如果 Cookie 被刷新，也会尽量自动回写配置。

如果登录失效，动态轮询会暂停。重新登录后会恢复。

## 常用配置

大多数情况下保持默认即可。

- `轮询间隔`：多久检查一次动态和直播状态，默认 30 秒。
- `接口请求间隔`：连续请求 Bilibili 接口之间等待多久，默认 0.5 秒。
- `直播检测`：是否推送开播和下播。
- `关注分组名称`：自动关注时加入哪个 Bilibili 关注分组。
- `补发时间窗口`：启动时是否补发最近动态，默认 0，不补发。
- `风控暂停时长`：检测到请求被拦截后暂停多久。

不建议把轮询间隔设置得很短。Bilibili 接口访问越频繁，越容易触发风控。

## 订阅

在主程序后台添加发布者时，可以输入：

- Bilibili UID。
- UP 主用户名。

搜索到发布者后，选择要推送到的 QQ 群或好友即可。

如果开启了主程序的“订阅时自动关注”，插件会尝试自动关注该 UP 主。

## 链接解析

支持常见 Bilibili 链接：

- 动态链接。
- 视频链接。
- 直播间链接。
- 用户空间链接。
- b23.tv 短链接。

如果链接解析模板里包含视频占位符，视频链接会尝试下载并发送视频。视频大小上限为 0 时表示不限制大小。

## 使用建议

- 首次使用先完成登录，再添加订阅。
- 保持默认轮询间隔，除非确实需要更快检测。
- 如果订阅很多 UP 主，优先提高轮询间隔，减少请求量。
- 如果经常出现风控提示，增大请求间隔或暂停一段时间。
